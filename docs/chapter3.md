第3章 NeoRV32 处理器架构设计

3.1 整体架构设计

NeoRV32 处理器的设计目标明确指向教学与科研应用领域，其核心约束为硬件面积最小化。该设计目标决定了整体架构采用高度模块化的分层设计，并在功能完备性与资源占用之间寻求最优平衡。

系统整体架构由三个层次构成：处理器核心（CPU Core）、片上总线（SimpleBus）以及外设子系统。其中处理器核心实现完整的 RV32I 指令集，支持流水线深度的参数化配置；片上总线采用自定义的轻量级协议，在降低实现复杂度的同时满足基本通信需求；外设子系统包含可配置的存储器与标准外设接口。

架构的灵活性主要源于其参数化配置系统。通过 CoreConfig 与 SoCConfig 两个配置类，可在设计实例化阶段确定处理器的各项特性。CoreConfig 定义处理器核的关键参数：寄存器宽度（xlen）、是否启用 RV32E 精简寄存器集、乘除法扩展（M-extension）的启用状态以及流水线级数（1级、3级或5级）。SoCConfig 则管理外设层面的配置，包括 UART、定时器、GPIO 等外设的启用状态以及片上存储器的容量。该参数化机制基于 Chisel 的生成电路（Generator）能力，未启用的模块在生成的 RTL 代码中将被完全消除，从而实现真正的面积优化。

3.2 指令集实现

指令集架构的实现涵盖译码逻辑、立即数生成与运算单元三个主要部分。

译码器（Decoder）模块承担指令解析职责。该模块接收 32 位指令字，提取操作码、功能码及寄存器地址字段，输出统一的控制信号集。译码逻辑采用 Chisel 的 ListLookup 机制实现，通过模式匹配将指令匹配至对应的控制信号组合。相较于传统的逐位译码方式，该实现方式具有更强的可维护性与可扩展性，新指令的添加仅需在译码表中增加相应条目。

立即数生成器（ImmGen）负责处理 RISC-V 指令集中五种立即数格式的提取与重组工作。由于 RISC-V 采用分散式立即数编码（I-type、S-type、B-type、U-type、J-type），立即数位于指令字的不同位域。ImmGen 根据译码器输出的立即数类型选择信号，从指令字中提取对应位域并进行符号扩展或零扩展，生成 32 位立即数输出。

算术逻辑单元（ALU）接收两个操作数及运算类型标识，执行对应的算术或逻辑运算。操作数的选择由数据通路中的多路选择器完成，依据指令类型选择寄存器值、立即数或程序计数器值。分支跳转的判断逻辑亦集成于执行阶段，根据寄存器比较结果与分支类型决定程序计数器的更新值。

3.3 参数化流水线设计

流水线架构的设计采用参数化方法，支持单周期、三级流水与五级流水三种配置，以适应不同应用场景的性能与面积需求。

单周期模式采用组合逻辑实现指令的完整执行流程，指令从取指到写回在一个时钟周期内完成。该模式消除了流水线寄存器的开销，硬件面积最小，但关键路径较长，工作频率受限。该配置适用于对性能要求不高但追求面积极致优化的应用场景，亦为教学演示提供了最直观的数据通路模型。

三级流水线模式将指令执行流程划分为取指（IF）、译码执行（ID/EX）与访存写回（MEM/WB）三个阶段。该配置在寄存器开销与性能提升之间取得平衡，通过减少组合逻辑级数提升工作频率，同时控制流水线寄存器数量。

五级流水线模式采用经典的 RISC 流水线划分：取指（IF）、译码（ID）、执行（EX）、访存（MEM）与写回（WB）。该配置最大化地利用了指令级并行性，但需解决数据相关与控制相关引发的流水线冒险。设计中在各流水线级之间设置了握手机制，通过 stall 与 flush 信号实现流水线的暂停与清空，为后续实现数据前递（Forwarding）与动态分支预测提供了扩展接口。

各级流水线之间的数据传递通过定义统一的 Bundle 结构实现。IFIDBundle、IDEXBundle、EXMEMBundle 与 MEMWBBundle 分别封装对应流水线级间需要传递的信号，确保接口的一致性与类型安全。

3.4 存储系统与总线

存储子系统基于 SimpleBus 自定义总线协议构建。该协议定义了请求-响应握手机制，包含地址、写使能、写数据、字节掩码等请求信号，以及读数据和有效指示等响应信号。该协议为同步总线，适用于单主多从的片上互连场景，未实现突发传输、乱序响应或缓存一致性等复杂特性。

片上存储器（OnChipRAM）通过 SimpleBus 挂载于总线，支持按字节、半字、全字（Byte、Half-word、Word）的访存操作。存储器支持初始化数据的加载，可在仿真阶段通过文件导入程序代码，或在综合阶段嵌入初始化数据。该特性为软件测试的快速迭代提供了便利。

3.5 外设子系统

外设子系统包含定时器（Timer）、UART 串口控制器与 GPIO 通用输入输出接口。定时器作为基础外设默认启用，实现 64 位自由运行计数器，为软件提供时间基准。UART 控制器实现标准串行通信接口，支持调试信息的输出与交互式调试。GPIO 接口提供通用数字引脚，可配置为输入或输出模式。

所有外设均采用内存映射寄存器（Memory-Mapped I/O）方式进行访问，软件通过访问特定地址范围内的存储器地址实现对外设的读写操作。总线译码逻辑依据地址范围将请求路由至相应的外设或存储器。
